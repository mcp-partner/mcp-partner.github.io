Introduction

MCP Partner is a client-side debugging and interaction tool for the Model Context Protocol (MCP). Think of it as "Postman for MCP". It allows developers to connect to MCP servers, discover available tools, and execute them with an intuitive UI.

Quick Start

1

Connect to a Server

Enter the URL of your MCP server in the top bar. The application supports standard SSE (Server-Sent Events) and Streamable HTTP endpoints.
Example: http://localhost:3000/sse

2

Select a Tool

Once connected, the sidebar will populate with available tools discovered from the server. Click on any tool to view its definition.

3

Execute & Debug

Fill in the parameters using the auto-generated Form view or switch to JSON view for raw editing. Click Send to execute the tool and view the JSON-RPC response in real-time.

Key Features

Dynamic Forms

Automatically generates UI forms based on the tool's JSON Schema, supporting validation, booleans, and descriptions.

Request History & Logs

View full JSON-RPC traffic in the bottom console, including request IDs, methods, and full error stacks.

State Persistence

Your server list, tool parameters, and execution results are saved locally, so you can pick up where you left off.

Config Import/Export

Share your server configurations with teammates by exporting a simple JSON file.

CORS & Proxying

Browsers enforce strict CORS (Cross-Origin Resource Sharing) policies. If your local MCP server does not send the correct Access-Control-Allow-Origin headers, the browser will block the connection.

Symptoms of CORS issues: Connection failing immediately with "Network Error" or 0 status code logs.

Solutions

  1. Use the built-in Proxy: Click the üõ°Ô∏è icon in the connection bar and enable "Use Proxy". This routes requests through a CORS-friendly forwarder.
  2. Configure your Server: Ensure your MCP server implementation sets Access-Control-Allow-Origin: * (or your specific domain) for both SSE and POST endpoints.
  3. Use Pancors: For heavy local development, run a local proxy like pancors.

Supported Transports

SSE (Server-Sent Events)

The default mode. The client connects via GET to receive events and discovers a POST endpoint for sending requests.

Streamable HTTP

Uses the newer MCP SDK transport method over HTTP streams. Useful for environments where standard SSE might be restrictive or for specific server implementations.
